# 規格說明書：提升測試覆蓋率至 80% 以上

## 概述 (Overview)
本任務旨在全面提升 `slctl` 專案的測試覆蓋率。目前部分 Package 的覆蓋率尚未達到專案要求的 80% 門檻，我們將透過新增單元測試與集成測試，確保核心邏輯與命令處理器（Command Handlers）皆有足夠的測試保護，以提升代碼穩定性與可靠性。

## 功能需求 (Functional Requirements)
- **覆蓋率提升**：針對所有目前測試覆蓋率低於 80% 的 Package 進行補強。
- **測試範疇**：
    - 涵蓋 `pkg/` 下的核心邏輯。
    - 涵蓋 `cmd/slctl/` 下的 CLI 命令處理邏輯。
    - 確保包含成功案例、邊界條件以及錯誤處理路徑。
- **驗證機制**：必須產出全專案的覆蓋率報告 (`coverage.out`)，並確認最終數值超過 80%。

## 非功能需求 (Non-Functional Requirements)
- **代碼風格**：遵循專案既有慣例，使用 Go 標準庫的 `testing` package 撰寫測試，不引入額外的斷言庫（Assertions Library）。
- **可維護性**：測試代碼應保持簡潔且具備良好的可讀性。
- **執行效能**：測試套件應能在合理時間內執行完畢，避免過度依賴耗時的外部網路請求（應使用 Mock 處理）。

## 驗收標準 (Acceptance Criteria)
- [ ] 全專案各 Package 的測試覆蓋率均達到或超過 80%。
- [ ] `go test ./...` 命令執行通過，無任何失敗測試。
- [ ] 產出最終的覆蓋率彙整報告，證明已達成目標。

## 超出範圍 (Out of Scope)
- 本任務不包含對既有功能的邏輯重構，除非是為了提高測試性（Testability）而必須進行的微調。
- 不包含效能測試或壓力測試。
